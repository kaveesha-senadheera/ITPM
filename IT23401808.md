# Singlish to Sinhala Translator - Automated Testing

Automated testing framework for the Singlish to Sinhala translator web application using Playwright.

## 📋 Overview

This project automates the testing of the [PixelsSuite Chat Translator](https://www.pixelssuite.com/chat-translator) by:
- Reading test cases from an Excel file
- Automatically typing Singlish input into the translator
- Capturing the Sinhala output
- Comparing with expected results
- Writing results back to Excel with PASS/FAIL status

## 🚀 Features

- ✅ Automated browser testing with Playwright
- ✅ Excel-based test case management
- ✅ Support for merged cells in Excel
- ✅ Automatic retry logic for slow translations
- ✅ Periodic saving of results
- ✅ Screenshot capture on errors
- ✅ Headless and visible browser modes
- ✅ Configurable wait times and retries

## 📊 Excel File Format

The Excel file should have the following columns:

| Column | Name | Description |
|--------|------|-------------|
| A | Test Case ID | Unique identifier (e.g., Pos_Fun_0101) |
| B | Test case name | Description of the test |
| C | Input length type | S (Short), M (Medium), L (Long) |
| D | Input | Singlish text to translate |
| E | Expected output | Expected Sinhala translation |
| F | Actual output | **Auto-filled by script** |
| G | Status | **Auto-filled by script** (PASS/FAIL/COLLECTED/ERROR) |

### Status Values
- **PASS**: Actual output matches expected output exactly
- **FAIL**: Actual output differs from expected output
- **COLLECTED**: Output captured but no expected value to compare
- **NO OUTPUT**: Translation did not produce any output
- **ERROR**: An error occurred during testing

## 🎯 Usage

### Basic Usage

```bash
python test_automation.py
```

### With Custom Excel File

```bash
python test_automation.py --excel "Assignment 1 - Test cases_IT23406308.xlsx"
```

### Headless Mode (No Browser Window)

```bash
python test_automation.py --headless
```

### With Periodic Saving

```bash
python test_automation.py --save-every 5
```
This saves results after every 5 test cases.

### Keep Browser Open After Testing

```bash
python test_automation.py --keep-open
```

### Custom Wait Times

```bash
python test_automation.py --wait-ms 10000 --retries 20
```

## ⚙️ Command Line Options

| Option | Default | Description |
|--------|---------|-------------|
| `--excel` | Assignment 1 - Test cases_IT23406308.xlsx | Path to Excel file |
| `--sheet` | " Test cases" | Sheet name in Excel |
| `--url` | https://www.pixelssuite.com/chat-translator | Translator URL |
| `--headless` | False | Run browser in headless mode |
| `--wait-ms` | 8000 | Initial wait time after translation (ms) |
| `--retries` | 15 | Number of retry attempts for output |
| `--retry-wait-ms` | 800 | Wait time between retries (ms) |
| `--save-every` | 0 | Save after N test cases (0 = save only at end) |
| `--keep-open` | False | Keep browser open after tests |
| `--timeout-ms` | 60000 | Page load timeout (ms) |
| `--type-delay-ms` | 30 | Delay between keystrokes (ms) |
| `--slow-mo-ms` | 0 | Slow down browser actions (ms) |

## 🔧 Troubleshooting

### Issue: "Permission denied" when saving Excel
**Solution**: Close the Excel file before running the script.

### Issue: "Could not find Chat UI locators"
**Solution**: 
- Check internet connection
- Verify the translator website is accessible
- Try increasing `--timeout-ms 120000`

### Issue: Tests show "NO OUTPUT"
**Solution**: 
- Increase wait time: `--wait-ms 10000`
- Increase retries: `--retries 20`
- Check if website is working manually

### Issue: All tests show "FAIL"
**Solution**: 
- Check if expected output in Excel matches translator output exactly
- Verify Sinhala Unicode characters are correct
- Run with `--keep-open` to manually verify output

### Issue: Script is too slow
**Solution**:
- Use `--headless` mode
- Reduce `--type-delay-ms 10`
- Reduce `--wait-ms 5000` (but may cause missed outputs)

## 🔄 Workflow

1. **Prepare Excel file** with test cases
2. **Close Excel file** (important!)
3. **Run script**: `python test_automation.py --headless --save-every 5`
4. **Monitor progress** in terminal
5. **Open Excel file** to view results
6. **Review failures** and update test cases if needed

## 📈 Performance Tips

- Use `--headless` for faster execution
- Set `--save-every 10` to balance between safety and speed
- Adjust `--wait-ms` based on your internet speed:
  - Fast connection: `--wait-ms 5000`
  - Slow connection: `--wait-ms 10000`
- Use `--type-delay-ms 0` for instant typing (may cause issues)
